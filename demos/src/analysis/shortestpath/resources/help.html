Description

yFiles for Java (Swing) is shipped with its own armada of graph analysis algorithms to attack problems to efficiently visualize your data. One of these is the shortest path search which is visualized in this small demo.

Things to try

  • Mark source and target nodes by selecting a node, then clicking on the Mark as Source or Mark as Target button. Select multiple nodes by a marquee selection with the mouse, or use the Ctrl/Command key when clicking on a node to add it to an existing selection.
    • The shortest path is calculated and marked in red as soon as at least one source and one target are selected.
    • The shortest path is the one with the smallest cost, where cost is the sum of the visible lengths of the edges.
  • Use Uniform Edge Labels to give each edge equal weight specified in the text field. The shortest path becomes the one where the sum of the edge weights is the smallest, i.e. the one with the fewest edges.
  • Press F2 to open the label editor for a selected edge and change the weight to see how the shortest paths are affected.
  • Toggle between Directed and Undirected edges to either taking edge direction into account or ignoring it when performing a shortest path search.
  • Edit the graph by creating, deleting, or moving nodes, edges, and bends. Observe how paths are affected. Edits immediately trigger a new shortest path search.
  • Select different automatic layout styles from the drop-down in the toolbar and observe how the changed edge lengths affect the shortest path search.
