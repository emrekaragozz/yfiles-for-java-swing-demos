Step 12: Bind Custom Data to Elements

The usual method to bind arbitrary custom data to graph elements is to add an IMapper to the graph's MapperRegistry. In this demo the creation time of each node is stored in this way.

Another approach is to use the ITagOwner interface of the IModelItem to bind a single datum to a graph element.

Display tool tips for these data items and a popup menu to manipulate them.

Note: If folding is enabled, like in this sample, we recommend to add the mapper to the MapperRegistry of the master graph. Folding then makes sure that a corresponding mapper is available for the graph of the managed view; the data can be accessed with both the master elements and the elements of the managed view.

Persisting the custom data is shown in a later step.
