Description

This demo shows the usage of ports by taking the example of a digital system consisted of logic gates.

Each node in this system has specific ports for incoming and outgoing edges.

Incoming edges are connected only to green ports (usually on the left side of the target node). Outgoing edges connect to blue ports (usually on the right side of the source node).

To achieve this, the notions of PortConstraint and PortCandidate have been used.

Things to try

  • Drag nodes from the palette. Notice that the new nodes already have green and blue ports.
  • Hover over a node to see which ports are valid (highlighted with green color) and invalid (highlighted with red color) to start an edge creation.
  • Create a new edge by dragging from a valid port. While creating the connection, the valid and invalid end ports are indicated with green and red color again. The edge will be created in the direction that has been selected from the Edge Direction Policy:
    • Start at source: Edges can only be created from source (blue) to target (green) ports.
    • Start at target: Edges can only be created from target (green) to source (blue) port.
    • Keep direction: Edges will be created in the same direction as last time.
    • Determine from port candidates: Edges will be created in the direction depending on whether the creation started at a source or target port.
  • Re-assign a connection by selecting the edge and dragging one of its endpoints.
  • Apply a new HierarchicLayout or Route Edges and notice that the ports remain unchanged.
