Description

Demonstrates how to implement a custom INodeStyle that displays Scalable Vector Graphics. The BATIK library is used for handling the required (low-level) SVG rendering.
Additionally, this demo shows how to configure yFiles for Java (Swing)'s GraphMLIOHandler to embed displayed SVG documents in GraphML.

Things to try

  • Zoom in and out and/or resize nodes. Unlike raster images, the quality of the displayed vector graphics will not diminish when changing zoom levels or node sizes.
  • Move nodes around with double-buffering enabled and double-buffering disabled to get an impression of the performance improvement gained through double-buffering.
  • Save a displayed graph and inspect the generated GraphML file.
